# 商用空调智能化产品规划（2026年H2）

1. # 五看-战略分析

## **看行业 / 趋势**

- **行业竞争正在从单机设备能力、平台单点能力、转向系统级能源成本运营能力的竞争。**

- 头部厂商在展台上呈现出的光伏、多能互补、储能联动、削峰节费、低碳运营等内容，头部暖通厂商正在加快从“设备制造商”向“综合能源解决方案提供方”延展；未来比拼的不仅是设备与平台的智能化程度，而是是否具备围绕客户能源成本、运行效率和低碳目标进行整体优化的能力，这意味着暖通行业正在从 **设备产业向项目运营产业转型。**

- 平台的价值不再只是“把设备接进来、设备可远程控制、把数据展示出来”，而是要解决两个个核心问题：
  
  - **能不能形成能源管理闭环**：客户更关心的是项目落地后能否持续节能、能否稳定运维、能否在改造场景中快速交付并形成可验证的结果；需要支持**从监测、诊断、优化、控制、验证审计五个关键环节**，形成完整链路；
  
  - **能不能把项目经验沉淀成可复制能力或解决方案**：未来几年的行业机会，不在于再做一个“功能更多的平台”，而在于把平台做成一套**可交付、可验收、可复制**的业务底座与行业解决方案。

- **平台生态与开放集成能力成为大型项目门槛，** BACnet、Modbus、MQTT、OPC UA等楼宇与工业协议，并且需要支持第三方品牌设备（暖通、照明、热水等）支持**统一接入**

- **AI开始进入实际应用阶段**，AI 不再只是“数据分析工具”，而是逐步参与系统控制策略。

## **看市场 / 客户**

- **旧改项目市场**：中国大量商业建筑中的暖通系统已经运行 10‑20 年，但整体设备并未到完全报废阶段，相比整体更换设备，客户更愿意选择：**通过数字化改造降低能耗，实施周期更短，ROI 更容易量化，投资规模更少，因此存量改造很可能成为未来几年最确定的增长市场，这要求厂商需要具备以下能力：**
  
  - 异品牌设备兼容与统一接入管理
  
  - 不停机改造（如：换外机不换内机，外机加装云网关或末端智控网关）
  
  - 尽可能无线接入，少施工、少扰动、快速上线

- 大型项目下，需要支持多品牌设备支持**统一接入智能控制与可量化项目能源管理与项目节能（设备统一集控管理与统一能耗管理），**大型建筑项目很少只使用一个品牌的系统，如果平台无法接入异构设备，很难在实际项目中落地；因此未来平台竞争很大程度上会取决于：**谁能更低成本更便捷地接入复杂业务系统与多品牌设备。**

- 对**持续运营降本**与**可验收的节能成果**的诉求强于单次设备参数对比；相比一次性采购更看重后续运营阶段是否能持续降本，是否能看到清晰、可量化的收益

## 看自己

- **已有能力**：中小型设项目与设备治理、工单售后安装、权限与多租、组态与数据看板等**工程化链条相对完整的能力已具备；基础的能耗检测能力已具备，含：****分户计费、能耗统计模块、电表管理模块**

- **能力缺口**：
  
  - 能源管理未能完全闭环（**“监测（已支持）—诊断—优化—控制—验证”的完整闭环**）
  
  - 节能手段单一，仅支持单机节能算法，缺乏多样化的节能优化策略；节能能力尚不足以支撑对外对赌/节能绩效类（EMC 合同）的商务机会
  
  - 中小型项目场景下的光伏/储能协同接入能力与数据缺失。
  
  - 旧改项目能力（接第三方室内机实现智能控制）
  
  - 螺杆机产品已推出，但配套的选型工具、高效机房、数据中心智能化解决方案缺失。
  
  - 组态可视化平台定制化程度高，交付周期较长。
  
  - 缺乏AI场景应用案例作为GTM营销抓手。

## **看机会**

- **构建可验收的能源管理业务闭环：即，****监测（已支持） → 过耗与异常归因诊断 → 优化策略与控制 → 能源审计，需要从单机节能到系统运营能效与可验收结果**

- **扩充节能优化策略**：结合地区气象、区域实时电价、光伏发电、储能状态、冷热负荷需求联动形成多样化的节能策略。

- **AI聚焦解决具体业务场景问题，先深后广**（结合openclaw能力或打造场景Agent，如：AI负荷/能耗预测；控制类 AI Agent与 分析类AI Agent、AI组态快速编排交付）适合作为中长期能力储备，以及GTM市场传播中的辅助亮点

- 支持第三方设备与平台系统快速接入，支持旧项目智能化改造方案与旧改项目交付产品化

- 在数据机房场景与大水机项目中的核心业务链路，含：售前选型（选机型、算数量、测算COP、计算项目ROI）→设备安装调试→运行优化→机房维护；其中【售前选型】环节可作为业务切入点增加智能化的选型与项目测算工具。

2. # 三定-战略落实

## 定战略控制点（构建核心壁垒/竞争力）

- **“可验收的节能结果”作为第一战略控制点**：平台能力的核心不再是“看得见、管得到”，而是围绕项目实际能源成本下降与运行效率提升，形成“监测—诊断—优化—控制—验证”的完整闭环，并能够输出可量化、可复盘、可验收的节能成果。

- **“第三方设备接入 + 旧改低扰动交付”作为第二战略控制点**：持续构建 BACnet、Modbus、OPC UA 等标准协议接入能力，并沉淀第三方品牌设备适配库、点位模板库和统一物模型能力。围绕“换外机不换内机”“加装网关实现远程运维”等高频旧改需求，沉淀标准硬件、标准方案、标准施工与标准验收模板，将旧改从项目制能力逐步做成产品化交付能力。

- **重点场景智能化解决方案标准化复制能力作为第三战略控制点**：围绕四大行业沉淀“方案包 + 交付包 + 运营包 + 案例包”，将单个项目经验转化为可复制的解决方案，减少每个项目都从头定义的成本。

- **“大水机售前测算工具化能力”作为第四战略控制点**：面向销售、售前与方案团队建设智能化选型与测算工具，打通“项目参数录入 → 机组选型 → 能耗测算 → ROI 评估 → 方案书输出”的前链路能力，提升大项目标前支撑能力，并带动设备销售与方案竞争力

- **“AI显性化应用 ”作为第五战略控制点**：AI 不作为独立卖点存在，而是嵌入已有的业务中，形成客户可感知、销售可讲述、交付可落地的“AI ”营销卖电，用于提升效率与市场传播。

## 定战略/业务目标（明确业务方向与指标）

- 打造「能源运营&能源治理」的综合解决方案，**年底前完成1～2 个能源运营**闭环标杆项目并可对外演示

##### **节能优化策略**

- **AI节能日程优化策略** 根据项目营业时间、节假日、办公时段、开关机历史行为数据等信息，生成Ai节能日程，减少非必要时段运行。

- **削峰填谷与需量控制策略** 结合实时电价、峰谷时段、历史负荷数据，控制高峰时段的负荷水平，并将可转移负荷向低谷时段迁移，降低综合电费成本。

- **负荷预测驱动策略** 基于天气、历史运行数据、时段特征与项目属性进行短时负荷预测，提前调整设备运行状态，提高系统响应效率并减少过度运行。

- **源网荷储协同策略** 对接光伏、储能及充电负荷，在满足项目运行需求的前提下，实现“光伏优先自用、储能削峰填谷、负荷柔性调节”的协同优化，并且在综合态势页面展示通过光伏储能协同产生的能源节省量。

#### P1 AI 场景应用

##### AI 组态助手

- **入口：**T-Service平台、IOC组态可视化平台

- **AI组态快速编排交付**：
  
  - 通过自然语言输入项目需求，AI 自动完成页面草稿生成、组件配置、布局排版、文案填充、图表选择、点位绑定、交互设计等工作，帮助用户以更少时间完成组态页面搭建与调整，包括：可根据用户描述的需求，快速生成一个组态草稿；根据设备类型和点位需求自动推荐适合的图表组件

- **组态AI数字人讲解**：AI数字人支持讲解与演示组态项目大屏，支持通过自然语言解答用户对于组态大屏的疑问
  
    面向销售、市场、客户演示、项目汇报场景；AI数字人基于组态大屏自动生成固定讲解路径的通用型脚本，对项目大屏内容进行口播演示，并支持用户围绕组态大屏特定范围或特定问题进行自然语言语音播报问答。
  
    典型场景包括：
  
  - 客户参观展厅时，对组态大屏进行自动讲解
  
  - 展会现场自动演示某个样板项目的运行情况
  
  - 售前汇报中辅助讲解能源驾驶舱、设备监控页、节能成果页
  
  - 客户询问“这张图是什么意思”“为什么这个项目今天用电较高”“这个设备现在状态如何”时，系统可直接应答

##### AI 项目管家（管理项目能源、控制）

AI Agent / OpenClaw 在本方案中主要承担“自然语言交互层 + 业务编排层 + 解释生成层”的角色，不可替代底层的数据采集、权限校验与控制执行能力。诊断、策略建议、报告生成等场景适合以 AI Agent 为主入口

- **入口：**T-灵控App、T-Building平台、飞书、钉钉、企业微信等

- **集控Agent**：设备/区域批量控制、单台精准控制、执行结果统计与二次确认；

- **日程Agent**：设备日程与项目级日程的创建、展示和触发逻辑

- **节能日程Agent**：AI根据项目营业时间、节假日、办公时段、开关机历史行为数据等信息，生成Ai节能日程，减少非必要时段运行。

- **能耗报告与归因辅助Agent：**
  
  - 项目能耗复盘：查看系统数量、设备规模、总耗电、电费预估、主要耗电系统、开机时段分析、系统级耗电排行、设定温度合理性分析、舒适温度区间分析。
  
  - 节能诊断：识别月/季度/年度“最不节能”的行为，如：疑似忘关机、设定温度不合理
  
  - 异常设备定位：从项目分析下钻到**具体内机**，查看哪台设备问题最严重。
  
  - 节能策略优化建议：提供优化建议，节能收益测算、生成AI节能日程、结合当地即时气候数据的温度锁定建议、即时集中控制策略建议。
  
  - 节能前后对比、可生成月/季/年度报告

- **设备管家Agent:**AI 查询与状态监测：设备状态查询（运行/故障/参数）、项目全局统计（实时开关机/离线数）

**注：控制类 AI 需要有约束**

- 所有通过AI Agent下发的指令必须经**角色权限校验**；AI Agent下发的指令支持全量**操作流水**；平台支持**一键回退到AI执行的规则策略/日程**。

- 与已有的分户计费「欠费停机」等现有业务逻辑**兼容**，避免 AI 绕过平台其他业务板块的原规则。

#### P2 大水机项目智能化选型与项目测算工具

面向售前、方案、销售与交付团队，建设覆盖“售前选型—方案测算—投标输出—交付衔接—运行优化”的智能化工具，增强大水机项目的前链路能力。

- **项目基础参数录入** 支持录入项目所在地、建筑类型、面积、负荷需求、运行时段、设计工况、冷冻/冷却水参数、电价规则、机房约束条件等基础信息。

- **机组选型推荐** 根据项目工况与产品参数库，自动推荐适配机型、台数组合、冗余配置与控制方式，支持不同方案对比。

- **系统能效测算** 结合典型气象年、设备性能曲线与负荷分布，对机房综合能效、全年耗电量、峰值需量、季节能效表现进行测算，形成更有说服力的售前依据。

- **经济性测算** 输出初投资、运行成本、节电收益、回收周期、ROI等指标，支持从“卖设备”转向“卖结果、卖方案”。

- **方案对比与自动生成报告** 自动输出多方案对比报告、设备清单、主要参数表、节能测算说明、项目 ROI 报告等要点，为销售和售前减少重复劳动。

- **运行优化闭环衔接，沉淀行业专用场景数据模板** 项目落地后将实际运行数据回流到选型模型中，对测算准确性进行校正，逐步形成项目经验数据资产，扩展到数据机房、医院、园区等复杂行业场景，形成行业专用模板。

**价值**

- 提升售前方案响应速度

- 提高投标方案专业度与可信度

- 强化设备销售转化能力

- 为后续高效机房业务奠定前端抓手

#### P3 支持第三方设备与平台系统快速接入，支持旧项目智能化改造方案与旧改项目交付产品化

- **统一接入底座建设** 建立统一第三方接入能力，支持 BACnet、Modbus、MQTT、OPC UA 等主流协议，并沉淀设备驱动库、品牌适配库、点位模板库和统一物模型映射能力，减少项目级重复开发。
